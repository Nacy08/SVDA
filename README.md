# SVDA: ValueRank & TwoStageDPO

[中文](#中文说明) | [English](#english)

## 中文说明

本仓库包含我们参加 **NLPCC 2026 Shared Task 2: Schwartz's Basic Human Values Detection and Alignment with LLMs (SVDA)** 的代码。任务包括细粒度价值检测与指定价值回复生成；对应系统分别为 **ValueRank** 和 **TwoStageDPO**。

### 方法概览

- **Track 1 — 价值检测：** 以 RoBERTa-large 为基础，结合 Schwartz 相邻价值排序约束、R-Drop、FGM、EMA 和三模型软投票。
- **Track 2 — 价值对齐生成：** 以 Meta-Llama-3.1-8B-Instruct 为基础，依次执行 4-bit QLoRA-SFT、候选重排构造偏好对、第一阶段 DPO，以及面向困难价值类别的计划引导第二阶段 DPO。

### 官方测试结果

| 任务 | 系统 | 主要结果 | 排名 |
|---|---|---|---:|
| Track 1 | ValueRank | Accuracy 93.00%，F1 93.02% | 3 |
| Track 2 | TwoStageDPO | Win 20.04%，Tie 72.57%，Lose 7.39% | 3 |

### 目录结构

```text
track1/                    # ValueRank：预热、RoBERTa 微调与集成
track2/
  sft/                     # QLoRA-SFT
  stage1_dpo/              # 第一阶段 DPO
  stage2_plan_dpo/         # 计划生成、困难样本构造与第二阶段 DPO
```

运行前请根据本地环境修改 YAML 和命令行中的模型、数据及输出路径。数据集、基础模型和训练权重未包含在本仓库中。Track 2 使用外部 LLM 进行计划生成、候选重排和验证评测；最终回复文本由微调后的策略模型生成。

## English

This repository contains our system for **NLPCC 2026 Shared Task 2: Schwartz's Basic Human Values Detection and Alignment with LLMs (SVDA)**. The two subtasks are fine-grained value detection and specified-value response generation, addressed by **ValueRank** and **TwoStageDPO**, respectively.

### Method overview

- **Track 1 — Value detection:** RoBERTa-large with a Schwartz-value neighborhood ranking constraint, R-Drop, FGM, EMA, and a three-model soft-voting ensemble.
- **Track 2 — Value-aligned generation:** Meta-Llama-3.1-8B-Instruct with 4-bit QLoRA-SFT, reranked preference-pair construction, first-stage DPO, and plan-guided second-stage DPO for difficult value categories.

### Official test results

| Task | System | Main results | Rank |
|---|---|---|---:|
| Track 1 | ValueRank | Accuracy 93.00%, F1 93.02% | 3 |
| Track 2 | TwoStageDPO | Win 20.04%, Tie 72.57%, Lose 7.39% | 3 |

### Repository layout

```text
track1/                    # ValueRank warm-up, RoBERTa fine-tuning, and ensemble
track2/
  sft/                     # QLoRA supervised fine-tuning
  stage1_dpo/              # First-stage DPO
  stage2_plan_dpo/         # Planning, hard-case construction, and second-stage DPO
  response_reranking/      # Candidate reranking and pairwise evaluation
```

Before running the code, update model, dataset, and output paths in the YAML files and command-line arguments. The dataset, base models, and trained weights are not included. External LLMs are used for planning, candidate reranking, and validation-time evaluation in Track 2; response texts are generated by the fine-tuned policy model.
